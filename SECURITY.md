# Security policy

## Supported version

Security fixes are made on the `main` branch. No released version currently
receives separate security updates.

## Reporting a vulnerability

Do not open a public issue for a vulnerability. Use the repository's
[private vulnerability reporting](https://github.com/i3sey/pipensx/security/advisories/new)
page and include reproduction steps, affected versions, and the expected
impact.

Never include Nintendo keys, credentials, copyrighted game data, or other
secrets in a report or test fixture.
