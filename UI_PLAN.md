# pipensx — план улучшения UI/UX

Цель: опыт максимально простой, интерфейс в духе системного UI Switch и оригинального eShop.
Аудит от 2026-07-02. Исходное состояние: весь UI в `src/main_switch.cpp` (~4150 строк),
46 захардкоженных `nvgRGB`, темы Borealis не используются, каталог из 6901 записи —
плоский список, все действия на скрытых кнопках (X/Y/LB/RB), тач-жестов и звуков нет,
i18n нет.

Задачи разбиты по сложности на три пула — **Fable** (сложные), **Opus** (средние),
**Sonnet** (простые). Внутри пула порядок = рекомендуемый порядок выполнения.
Зависимости указаны явно; задачи без зависимостей можно вести параллельно.

Палитра (фиксируем как токены): акцент `#00C3E3` (Joy-Con Neon Blue), ошибка `#FF4554`
(Neon Red), успех `#60DC82`, поверхности/текст — из тем Borealis light/dark.

### F6. In-memory кэш обложек — мгновенный повторный вход в каталог ✅ (выполнено)
Добавлено 2026-07-04. Сейчас `AsyncRgbaImage` при каждом создании ячейки заново
читает дисковый кэш и декодирует PNG/JPEG — после выхода из каталога и повторного
входа иконки «прогружаются» с нуля.
Объём: общий LRU-кэш в памяти (декодированный RGBA или живые NVG-текстуры) с
бюджетом ~64–128 MB поверх существующего дискового кэша; попадание в кэш —
синхронная подстановка в первый кадр, без мигания плейсхолдера; префетч
соседних строк грида и полок; инвалидация при обновлении каталога; аккуратная
жизнь текстур относительно NVG-контекста и потоков декодера.
Зависит от F2.
**Готово, когда:** повторный вход в каталог показывает уже виденные обложки в
первый же кадр, скролл по всем 6901 записям без роста памяти сверх бюджета
(замер на консоли).
**Выполнено 2026-07-05:** кэш живёт в `GameMetadataService`: существующий LRU
декодированного RGBA (`imageCache_`) поднят с 24 до 96 MB (середина окна
64–128 MB, бюджет держит LRU-вытеснение в `cacheImageLocked`). Синхронный
пробник `cachedImage()`: `loadImageInto` при хите кладёт NVG-текстуру в тот же
кадр (`AsyncRgbaImage::setRgbaNow`, UI-поток), минуя worker-очередь — без
мигания плейсхолдера. Префетч: `prefetchImage()` (декод без колбэка, коалесится
с обычными запросами) греет соседние грид-строки из
`CatalogDataSource::cellForRow` (`prefetchGridRow(row±1)`); полки/hero биндят
все свои карточки сразу и греются сами. Инвалидация: `dropMemoryImageCache()`
при успешном `refreshCatalog()` (дисковый кэш остаётся; очистка из Settings
чистит оба). Жизнь текстур не изменилась: кэш хранит immutable RGBA
(`shared_ptr<const DecodedImage>` из потоков декодера), NVG-текстурами
по-прежнему владеют сами вью. Тест `testImageMemoryCache` (PC): холодный
промах, префетч-прогрев, коалесс запроса на кэшированную копию, дроп.
Switch-сборка `pipensx.nro`, PC-тесты и golden зелёные; мгновенный повторный
вход и замер памяти по бюджету — при первом прогоне на железе.
